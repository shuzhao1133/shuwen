PART 01 · 配置 — Claude.ai 账号

路径一：Claude.ai 账号 SOP

访问 claude.ai，注册账号后升级 Pro（$20/月）。在浏览器或下载 Claude Desktop App 均可使用。注意：部分功能（Scheduled Tasks）仅在 Desktop App 中可用。

1
注册并升级 Pro
访问 claude.ai → 注册账号 → 左下角升级 Pro 计划。升级后可用 Sonnet 4.6 / Opus 4.7、Projects、Research、文件分析等全部 claude.ai 功能。
2
下载 Claude Desktop App（推荐）
从 claude.ai 下载桌面应用（macOS / Windows）。Desktop App 拥有浏览器版没有的功能：Cowork（文件夹自动化）、Scheduled Tasks（定时任务）、Dispatch（手机远程触发）。
3
按项目建立 Project
左侧边栏 → 「+ New project」，按项目名称创建。每个 Project 独立管理文件和对话，上下文隔离不串扰。
4
上传核心项目文件
在 Project 内上传：合同、方案、问卷、访谈记录、报告模板。命名建议：2024-访谈记录-成都-商户A.pdf
5
写入 Custom Instructions
Project Settings → Custom Instructions：设置报告风格、客户口径、禁止事项。Claude 每次对话都会遵守这些规则。
📋 Custom Instructions 模板（点击展开）▶
你是本项目的研究助手。项目背景：[四川省餐饮个体工商户动态监测研究]，委托方为[某省商务厅]。

输出风格：政府咨询报告风格，客观陈述，避免绝对化表述。
数据规范：政策数据必须标注来源和时间，不得编造统计数字。
格式：段落使用二级标题，要点用短句。

禁止事项：
- 不得将客户名称、项目编号写入任何输出
- 不得引用我未提供的"内部数据"或"调研结论"
- 所有政策摘引须注明"来源：[文件名]"
⚠ 文件安全：Project 内上传的文件会被 Claude 读取。禁止上传含客户姓名、手机、身份证、合同金额的原始文件。上传前脱敏处理。
PART 01 · 配置 — API Key

路径二：Anthropic API Key SOP

访问 console.anthropic.com 申请 Key。API 路径解锁无限制按 token 计费的灵活用量，配合缓存机制可节省 80%+ 成本。

1
申请 API Key
console.anthropic.com → API Keys → Create Key。命名建议加前缀：proj-sichuan-analysis。Key 只显示一次，立即复制保存。
2
安全存储——必须用环境变量
绝对禁止将 Key 写入代码或上传任何平台。正确做法：
macOS/Linux：echo 'export ANTHROPIC_API_KEY="sk-ant-..."' >> ~/.zshrc && source ~/.zshrc
Windows：系统设置 → 环境变量 → 新建 ANTHROPIC_API_KEY
代码中读取：import os; key = os.environ["ANTHROPIC_API_KEY"]
3
设置预算上限
console.anthropic.com → Billing → Spend limits：月度硬上限（如 $30）+ 软提醒（$20 时邮件通知）。
💡 缓存机制：节省 80%+ 成本（点击展开）▶
# 缓存前：100K tokens × $3/MTok = $0.30
# 缓存后：10K uncached × $3 + 90K cached × $0.30 = $0.057
# 节省：81%

# Claude Code 会话自动缓存 CLAUDE.md，无需额外配置
# API 调用中显式启用缓存：
response = client.messages.create(
  model="claude-sonnet-4-6",
  system=[{"type":"text","text":"...","cache_control":{"type":"ephemeral"}}],
  messages=[...]
)
⚠ 安全：API Key 等同账号密码。泄露后立即在 console.anthropic.com 撤销（Revoke）并重新生成。

PART 02 · Claude.ai — Research

Research：带来源的深度研究

这个功能在 Claude.ai 里叫 Research（不是 Deep Research）。它结合网页搜索、Google Workspace 和已连接集成，自主规划研究路径，多步检索后生成带来源引用的研究报告。

📍
在哪里找
新建对话 → 输入框左侧工具栏 → Research 图标（或选择 Research 模式）。需要 Pro/Max/Team/Enterprise 计划，Free 账号无此功能。
claude.ai → 新对话 → 工具栏 → Research
✅ 适用场景
• 政策梳理：某领域近三年政策演变
• 行业数据：市场规模、增速、区域分布
• 竞品 / 对标：其他地区典型案例
• 证据收集：可写入报告的可引用事实
• 支持 Google Workspace 文件检索
⚠️ 使用要求
• 必须要求"列出来源 URL 和发布时间"
• 区分"事实陈述"和"AI 判断 / 推断"
• 关键数据写报告前点进原始链接核实
• Research 结论不得直接写进正式报告
📝 Research 提示词模板（点击展开）▶

输出：政策名称 | 发布机关 | 发布时间 | 核心内容（1句）

2. 汇总行业规模数据（市场规模 / 商户数量 / 增速）
每条数据必须标注：来源机构 | 统计口径 | 发布时间 | URL

3. 整理四川省内典型案例（≥3个）
地区 | 案例名称 | 核心做法 | 来源

输出格式：
① 研究摘要（200字以内）
② 证据表（来源 | 核心信息 | 时间 | URL）
③ 可写入报告的核心观点（3-5条，区分"事实"和"判断"）

注意：不确定的内容请注明"待核实"，不得编造数据。
使用逻辑：Research 做广度检索 + 结构化汇总，人工做关键数据核实。AI 省去搜索整理时间，核实工作仍需人来做。
PART 02 · Claude.ai — 文件与数据分析

文件分析：上传即可读取、清洗、生成图表

直接上传 Excel、PDF、Word、CSV、图片，Claude 直接读取分析——生成统计表、交叉分析、图表和结论，无需提前写脚本。适合快速探索和初步分析。

📍
在哪里找
新建对话 → 输入框左侧回形针图标（Attach files）→ 上传文件后直接提问。或在 Project 对话中上传。
claude.ai → 任意对话 → 📎 上传文件
📊 问卷数据清洗五步流程（点击展开）▶
1
上传文件 + 说明背景
「这是问卷星导出的 Excel，A 列是编号，B-F 是李克特量表 1-5 分，G-H 是开放问答。」
2
要求先做数据质检
「先检查：① 有效行数，② 完全空白行，③ 量表列超出 1-5 的异常值，④ 重复编号。」
3
分组统计
「按 A 列行业分组，计算 B-F 均值，输出表格，说明各组差异是否显著。」
4
生成图表
「基于分组结果生成柱状图，蓝紫色系，每组标注样本量 n=xx。」
5
输出分析结论
「3-5 句描述核心发现，区分'数据事实'和'可能解释'，避免过度推论。」
数据安全：上传前确认无客户姓名、手机号、身份证等个人信息。问卷数据建议先在本地删除个人标识列。

Claude 的定时功能叫 Scheduled Tasks，在 Claude Desktop App 的 Cowork 标签里，不在浏览器版 claude.ai 里。它让 Claude 按你设定的时间自动执行任务并保存结果到本地文件夹。

📍
在哪里找（重要：必须用 Desktop App）
下载 Claude Desktop App → 顶部 Tab → Cowork → 左侧边栏 → Scheduled → + New Task。
注意：电脑必须开机并唤醒，任务才能运行。
Claude Desktop App → Cowork 标签 → Scheduled

写作与文档协作

Claude 在长文档写作和迭代上尤其出色——1M token 上下文意味着整份报告可以放在一个会话中反复修改，不再因上下文丢失而前后不一致。

📝 报告初稿四步法（点击展开）▶
1
提供结构框架
先给 Claude 报告大纲，让它理解整体逻辑再开始写作。
2
分章节生成，单独检查
「先生成第二章'现状分析'，基于以下数据点：...」每章独立生成、独立检查。
3
局部修改
「这段太学术化，改成口语化的政府报告风格」——针对段落而非全文重写。

PART 03 · Claude Code — CLAUDE.md

CLAUDE.md：项目规则文件

CLAUDE.md 是 Claude Code 的"宪法"——每次启动自动注入，告诉 AI 这个项目的数据口径、命名规范、禁止事项和输出格式。团队共享一个 CLAUDE.md，行为保持一致。

📍
在哪里找 / 创建
在项目根目录运行 /init，Claude Code 自动生成 CLAUDE.md 模板。手动创建也可以，文件命名必须是 CLAUDE.md，放在项目根目录。
项目根目录 / CLAUDE.md
⚠ 成本提示：CLAUDE.md 每次请求都会注入。保持 200 行以内，只写可验证的规则，不写"愿望"和"建议"。每多 1000 token，每次请求增加约 $0.003（Sonnet）的隐性成本。
📋 CLAUDE.md 完整模板（点击展开）▶
# CLAUDE.md — 四川餐饮个体工商户动态监测项目

## 项目背景
研究四川省餐饮类个体工商户的经营状态与政策支持。
数据来源：问卷调查（n=312）、政策文件、统计年鉴。

## 数据口径
- 个体工商户：工商部门登记的非法人主体
- 餐饮类：仅 GB/T 4754-2017 C 大类餐饮服务
- 时间口径：2021-2023 年为实际数据，2024 年为预估值（须标注）

## 构建与运行命令
- 数据清洗：python scripts/clean_data.py --input data/原始 --output data/清洗后
- 生成图表：python scripts/generate_charts.py
- 测试：pytest tests/

## 文件命名规范
- 原始数据：data/原始/[来源]_[日期].xlsx（只读，禁止修改）
- 清洗后：data/清洗后/[版本]_[日期].xlsx
- 输出图表：output/[类型]_[日期].png

## 禁止事项
- 禁止编造任何统计数据或政策内容
- 禁止将原始问卷的个人信息（姓名/手机）写入输出
- 禁止删除或覆盖 data/原始/ 目录下的任何文件
- 禁止在没有数据支撑的情况下生成"研究结论"

## 输出格式
- 统计表：xlsx，A1 注明数据来源和统计时间
- 图表：png，300dpi，含标题、轴标签、样本量标注
- 代码：每个函数含一行注释说明目的

## 并行限制
- 最多同时启动 3 个 subagent
- subagent 仅读取权限，编辑由主 agent 处理
最佳实践：把 CLAUDE.md 提交到 Git 仓库，团队共享同一份规则。专项任务的专用指令放进 Skills，不要塞进 CLAUDE.md。
PART 03 · Claude Code — Skills

Skills：封装可复用的任务能力

Skills 存放在 .claude/skills/<name>/ 下的 SKILL.md 文件 + 辅助脚本。它封装有真实域逻辑的任务，按需加载——不像 CLAUDE.md 每次都注入，只在被调用时消耗 token。

📍
在哪里创建
在项目根目录下创建 .claude/skills/技能名称/SKILL.md 文件。Claude Code 启动时自动发现所有 Skills。
.claude/skills/data-quality-check/SKILL.md
何时用 Skill vs CLAUDE.md vs Slash Command
CLAUDE.md：项目通用规则（口径、禁止事项、命名规范）

Skill：特定任务专用逻辑（有参数和辅助文件）

Slash Command：简单的提示词模板（无需参数和复杂逻辑）
适合封装为 Skill 的任务
• 问卷数据质检（缺失值/异常值检查）
• 标准分组统计（按行业/地区/规模）
• 热力图生成（参数化颜色和维度）
• Word 报告批量格式化
• 多文件合并去重
PART 03 · Claude Code — Subagents

Subagents：并行执行隔离任务

Subagent 是独立的 Claude 会话，有自己的上下文窗口。适合需要上下文隔离或并行加速的任务——但成本随并行数线性增加，必须谨慎使用。

Subagent 最适合的场景
• 同时处理 10 份访谈记录摘要提取
• 并行运行多个数据分析脚本
• 代码审查与功能开发同步进行
• 大型任务分解为独立子任务并行执行
⚠️ 成本控制要点
• 10 个并行 subagent = 10x token 消耗
• 在 CLAUDE.md 中限制最大并行数（建议 ≤3）
• 简单任务的 subagent 用 Haiku 降成本
• 任务完成后立即 /stop，避免空跑
⚠ 成本警示：Max 20x 计划才能支持大量并行。Pro 计划并行 3 个以上 subagent 会快速耗尽每日额度。
PART 03 · Claude Code — Plan Mode & 验收

Plan Mode、验收与回滚

Claude Code 会真实执行文件操作，养成"先规划再执行、执行后验收、出错能回滚"的三步习惯。

规划
Plan Mode：先看计划再执行
复杂任务前按 Shift+Tab 进入 Plan Mode。Claude 先分析项目结构、列出执行计划，你确认无误后再执行。
验收
三项必检
① 行数——输出行数与原数据是否一致；② 数值范围——统计量是否在合理区间；③ 文件可打开——格式是否正确、内容是否完整。
回滚
保留原始文件 + Escape 退出
运行前备份：data_原始.xlsx（只读禁止删除）。发现问题：Escape 两次中止，从原始文件重跑。Git 项目可用 git stash 回滚。
PART 03 · Claude Code — Routines

Routines：云端定时自动化

Routines 是 Claude Code 的云端定时任务（区别于 Cowork 的 Scheduled Tasks）：不需要电脑开机，在 Anthropic 服务器上运行，支持定时触发、Webhook、GitHub 事件多种方式。

📍
在哪里找
浏览器访问 claude.ai/code/routines，或在 Claude Code CLI 中用 /schedule 命令创建。需要 Pro/Max/Team/Enterprise 计划并开启 Claude Code。
claude.ai/code/routines  |  CLI：/schedule
触发方式说明典型用法
定时触发设定 cron 周期（每天/每周/每月）「每天早 9 点，汇总昨日政策新闻」
Webhook 触发外部系统 POST 到专属 URL表单提交 → 自动生成提案草稿
GitHub 触发响应 PR / Release 等 Git 事件每次 PR 自动触发代码审查
Routines vs Scheduled Tasks
Scheduled Tasks（Cowork Desktop）：
· 本地运行，访问本地文件
· 电脑必须开机
· 在 Desktop App → Cowork → Scheduled

Routines（Claude Code 云端）：
· 云端运行，访问配置的 Git 仓库
· 无需电脑开机
· 在 claude.ai/code/routines
项目组推荐用法
• 政策监测：每天早 9 点搜索最新政策动态
• 周报生成：每周五下午从 Git 仓库读取访谈记录，生成周报
• 代码审查：每次 PR 自动触发质量检查
• 数据同步：定期从外部系统拉取数据更新

合规与边界

合规使用规范：四条边界线

四条规则适用于所有成员使用 Claude.ai 和 Claude Code 的全部场景。不确定时先问项目负责人。

🚫 客户资料与涉密数据
  • 禁止将客户全名、手机、身份证、合同金额上传任何 AI 平台
  • 问卷数据上传前必须删除个人标识列
  • 标注"内部资料"或"保密"的文件禁止上传
  • 政府项目未公开数据禁止上传
⚠️ 上传文件安全规则
  • 上传前问："如果这个文件被公开，有什么后果？"
  • 优先上传摘要版本，而非完整原始文件
  • API Key、数据库配置文件禁止上传
  • CLAUDE.md 中不得写入任何密钥或真实客户信息
🚫 AI 幻觉核验要求
  • AI 生成的所有政策数据须点进原始链接核实
  • 统计数字（规模/增速/排名）须标注来源后人工确认
  • AI 生成的"引用"逐一核实是否存在原始出处
  • Research 结论不得直接用于正式报告
✅ 交付物复核规则
  • 含数据的交付物须有人工复核记录
  • AI 生成报告初稿须经项目负责人审阅后方可发客户
  • Claude Code 生成的 Excel/图表须与原始数据抽样比对
  • 政策建议须咨询专业判断后写入
总原则：AI 是助手，不是背书人。所有对外交付物的准确性、合规性和专业性，最终责任在项目团队。
总结

带走 5 个判断

不必把每个功能都用上。这 5 个判断帮你在面对新任务时快速决定用哪个工具。

01
日常工作用 Sonnet 4.6，复杂分析用 Opus 4.7，批量子任务用 Haiku
90% 的工作 Sonnet 4.6 都够用，成本只有 Opus 的 1/5。Haiku 用于大量简单的 Subagent 任务，可以节省 80% 成本。Fable 5 目前暂停，不要等它。
02
功能名称记清楚：深度研究叫 Research，定时任务在 Desktop App 的 Cowork 里
Research 在浏览器版 claude.ai 对话工具栏里；Scheduled Tasks 在 Claude Desktop App → Cowork → Scheduled，不在浏览器里。两者不要混淆。
03
CLAUDE.md 只写可验证规则，保持 200 行以内
每一行都是每次请求的成本。只写数据口径、命名规范、禁止事项——专项任务逻辑放进 Skills。
04
复杂任务必须用 Plan Mode，出错用 Escape 中止
Claude Code 会真实执行文件操作。Shift+Tab 进入 Plan Mode 先看计划，确认后再执行。发现方向错误立即 Escape 两次中止。
05
所有对外交付物必须人工复核，AI 不背书
Claude 负责提速、初稿、格式化——数据准确性、结论合理性、合规边界，永远是项目团队的责任。
